## Item 1 — F4 durable-resume steer

### Today
- Wire `Submit{...}` → `SessionApi::submit` (`node_api/session.rs:8`). It calls `self.claim(&session, Lifecycle::Live)?` **first** (`node_api.rs:465`). A session already owned by `Lifecycle::Durable` → `ApiError::Conflict`. An **unclaimed** id with a durable store row → `claim(Live)` succeeds and `self.live.submit` opens a **fresh live incarnation** over the child's id (does NOT hydrate the durable transcript). `submit_routed` (`session.rs:106`) does the same via `self.live.ensure(...)` at `session.rs:128`.
- The durable rail already exists and is proven: `inject_session_msg` (`node_api.rs:509`) routes a durable/parked session to `store.enqueue_session_input(session, cbor(UserMsg))` + `store.enqueue_wake(session)` (`node_api.rs:534-535`); the woken hydrate drains the pending input into the durable conversation. Live sessions fall through to `self.submit(...StartTurn...)`.
- Conformance `tests/daemon-conformance/src/node/operator_steer.rs` documents F4 as the recorded follow-up (module docs literally point at `.stream-plan.md`). `operator_assign_wakes_a_parked_durable_child` proves the store seam (`enqueue_session_input` → `Assign` wake folds `OPERATOR-WAKE-PING` into the parked child's re-checkpointed transcript while it stays `Suspended`).

### Design
"Parked-durable" = a session whose lifecycle is the **durable** one and whose durable status is live-but-dormant (`Active | Suspended{..} | Ready`), NOT `Completed`/absent. Detection mirrors `inject_session_msg`'s branch:
- `owners.get(session) == Some(Lifecycle::Durable)` → durable; or
- unclaimed (`None`) **and** `store.status(session)` is `Some(Active|Suspended|Ready)` → durable by evidence.

For a durable-evidenced session **and** command ∈ `{StartTurn, Steer}`, route through the durable rail instead of `claim(Live)` + fresh incarnation:
- `StartTurn { input, .. }` → enqueue `input` (the `UserMsg`).
- `Steer { text, .. }` → enqueue `UserMsg::new(text)`.
- then `store.enqueue_wake(session)`; return `Ok(())`.

Extract the durable-enqueue body of `inject_session_msg` into a private helper `enqueue_durable_input(&self, session, UserMsg) -> Result<(), ApiError>` (encode → `enqueue_session_input` → `enqueue_wake`, dropping on `Completed|None`) and reuse it from both `inject_session_msg` and the new submit gate. Add `fn try_durable_resume(&self, session, &AgentCommand) -> Option<durable-UserMsg>` that returns `Some(msg)` only for `StartTurn`/`Steer` on a durable-evidenced, non-settled session.

Call the gate at the **top** of `submit` (before `claim(Live)`); apply the same gate to `submit_from`, `submit_as`, and `submit_routed` (each opens live). Other commands (Snapshot/Observe/Cancel/…) are unchanged → a durable session still `Conflict`s for them (guard-rail intact).

### Why not a new op
Recorded + rejected in the item: Submit routing needs no client change and no wire bump. This keeps the wire `Submit` op the single entrypoint and reuses the proven store seam.

### Files
- `crates/substrate/daemon-host/src/node_api.rs`: add `enqueue_durable_input` + `try_durable_resume` helpers; refactor `inject_session_msg`'s durable branch onto `enqueue_durable_input`.
- `crates/substrate/daemon-host/src/node_api/session.rs`: gate `submit`, `submit_from`, `submit_as`, `submit_routed` on `try_durable_resume` before the `claim(Live)` / `live.ensure` path.

- `submit_steer_lands_in_the_parked_durable_transcript`: seed a parked durable child exactly like `operator_assign_wakes_a_parked_durable_child` (Suspended on an approval); call `node.submit(child, Steer{ text:"F4-STEER", .. })` (and a `StartTurn` variant); assert the text lands **inside** the durable transcript (peek snapshot conversation contains it) and the child stays `Suspended|Active` (the resume folds input, does not fast-forward the approval). ## Item 3 — Archived-children enumeration

### Today
`crates/substrate/daemon-host/src/node_api/roster.rs::session_in_scope` (`:406`):
```
SessionScope::Archived => i.role == SessionRole::Primary && i.archived,
```
So only archived `Primary` rows surface. `TopLevel`/`ByProfile`/`ByTransport` all exclude archived; `tree()` is a live-parent drill-down. An archived subagent child (`ManagedChild`/`EphemeralSubagent`) therefore has **no enumeration path**.

### Choice (justified)
Broaden `Archived` to match archived rows of **any role**:
```
SessionScope::Archived => i.archived,
```
Rationale: `Archived` is the explicit opt-in archived view (docs: "the default scopes exclude archived; this is the opt-in to see them"). Children that were archived (individually, or when a parent was archived) are otherwise unreachable — `TopLevel`/`ByProfile` filter them out and `tree()` only drills from live parents. Surfacing all archived roles makes the archived view complete without a wire change (the `SessionScope` enum is untouched; only the node-side filter changes). This is node-authoritative: the client already opts into `Archived` and renders whatever the node returns.

Considered + rejected: `role == Primary || role == ManagedChild` (would still hide `EphemeralSubagent`, arbitrary); a new scope variant (that IS a wire change — out of stream).

## Item 4 — Delegation-spawn ToolDetail

### Today

## Item 5 — CON-15 node half: descriptor-driven OAuth engine

Refactor `crates/adapters/daemon-oauth/src/lib.rs` from one RFC-shaped factory (endpoints arrive in client `begin.params`) into ONE descriptor-driven engine covering all variants. **No wire changes**: reuses existing `AuthApi`/`AuthProviderInfo`/`AuthParamField`/`AuthFlowKind`/`AuthBeginRequest`/`AuthBindRequest`; `AuthFlowKind::OAuth2Pkce` covers every variant (redirect carries `code`; `state` optional). `AuthOutcome`/`AuthFlowFactory`/`PendingAuthFlow` (host-internal, `daemon-host/src/auth.rs`) are NOT on the wire, so they may gain fields.

### 5.1 Descriptor type (`daemon-oauth`, new)
```rust
pub struct OAuthFlowDescriptor {
    pub authorization_endpoint: EndpointSource,   // Fixed(url) | FromParam(key)
    pub token_endpoint: EndpointSource,
    pub client_id: ClientIdSource,                // None | Fixed(String) | FromParam(key)  (PKCE public clients omit)
    pub client_secret: Option<SecretSource>,      // None | FromParam(key)                   (confidential clients)
    pub scopes: ScopeSource,                      // Fixed(String) | FromParam(key) | Empty
    pub callback_param: CallbackParam,            // RedirectUri  -> `redirect_uri=<cb>` (RFC 6749)
                                                  // CallbackUrl  -> `callback_url=<cb>`  (OpenRouter)
    pub use_state: bool,                          // append+enforce CSRF `state` (RFC); false = PKCE-only binding
    pub exchange: ExchangeStyle,
    pub credential: CredentialShape,              // how the outcome blob + slot are formed (see 5.4)
    pub identity: IdentitySource,                 // account label derivation (see 5.4)
    pub flow_kind: AuthFlowKind,                  // OAuth2Pkce for all current descriptors
    pub family: &'static str,                     // the auth_begin.family this descriptor serves
    pub display_name: &'static str,
    pub params_schema: Vec<AuthParamField>,       // EMPTY for provider-bound families
}

pub enum ExchangeStyle {
    /// RFC 6749 §4.1.3 form POST (grant_type=authorization_code, code, redirect_uri, client_id, code_verifier[, client_secret]).
    FormPost,
    /// A JSON POST body; `key_field` names the response field carrying the minted secret.
    JsonPost { body: JsonBodyShape, key_field: String },
}

/// The JSON exchange request body (OpenRouter: code + code_verifier + code_challenge_method).
pub enum JsonBodyShape { OpenRouterKeyMint }   // extensible; encodes the exact {code, code_verifier, code_challenge_method:"S256"}
```
PKCE (S256) is **always on** for every descriptor (verifier + challenge minted at begin regardless of `use_state`). `EndpointSource::FromParam` + `ClientIdSource::FromParam` reproduce today's generic `oauth2` family; `Fixed`/`None` express the curated providers.

### 5.2 One engine (`daemon-oauth`)
- `struct DescriptorFlowFactory { descriptor: Arc<OAuthFlowDescriptor>, http: EgressClient }` implements `AuthFlowFactory`:
  - `family()` → `descriptor.family`; `provider_info()` → `{ family, flow_kind, display_name, params_schema }` (empty schema for provider-bound families).
  - `begin(params, redirect_uri)`: resolve endpoints/client_id/scopes from the descriptor (Fixed or `require(params, key)`); mint PKCE verifier+challenge; mint `state` iff `use_state`; build the authorization URL — append `client_id`/`state`/`scope` only when present, `code_challenge`+`code_challenge_method=S256` always, and the callback under `callback_param` (`redirect_uri` vs `callback_url`), and `response_type=code` only for RFC descriptors (OpenRouter's authorize URL takes none). Return one `DescriptorPendingFlow` holding `{ http, authorization_url, token_endpoint, client_id, client_secret, redirect_uri, verifier, state:Option<String>, descriptor }`.
- `struct DescriptorPendingFlow` implements `PendingAuthFlow`:
  - `complete(callback)`: parse callback; enforce `state` echo **iff** `use_state` (skip cleanly when false — OpenRouter binds via PKCE only); extract `code`; run `exchange`:
    - `FormPost` → today's `EgressRequest::post_form(token_endpoint, pairs)` (RFC fields; `client_secret` when present), `Redirects::None`; blob = raw token JSON; identity per `IdentitySource`.
    - `JsonPost{ body, key_field }` → build the JSON body (`{code, code_verifier, code_challenge_method:"S256"}`), `EgressRequest::post_json(token_endpoint, &body)` (`daemon-egress` already has `post_json`, `lib.rs:126`), `Redirects::None`; parse response JSON, pull `key_field` (OpenRouter: `"key"`); the minted key becomes the **credential blob** per `CredentialShape` (see 5.4).
  - Build `AuthOutcome` per `CredentialShape` + `IdentitySource`.

The generic `oauth2` family becomes ONE `DescriptorFlowFactory` over a `descriptor::generic_oauth2()` (all `FromParam`, `use_state:true`, `FormPost`, `CredentialShape::RawTokenJson`, `IdentitySource::LabelThenSubThenHash`, non-empty `params_schema`) — **no behavior change**, no duplicate flow code. The current public consts (`FAMILY`, `PARAM_*`) are retained for the generic descriptor + tests.

### 5.4 Credential-ref / slot mapping (the crux)
How a **pasted** provider API key is stored today (traced):
- `credential_set(profile, secret)` → `CredentialStore::set(profile, secret)` (`node_api/cred_auth.rs:8`, `credstore.rs`). The store keys by an arbitrary string; for a model provider the key is the **profile id**: `build_owner_broker`/`build_multi_profile_broker` (`main.rs:3058`,`:3088`) bind a `PooledStoreCredentialSource` to `profile` and read `store.get(profile)`. The genai provider threads this as `Request.auth` per call (`genai_provider.rs` — the credential lease secret is the bearer). OpenRouter is a curated genai vendor (`DISCOVERY_ADAPTERS`, `genai_provider.rs:218`).
- ⇒ For a minted OpenRouter key to "ride the same downstream path as a pasted key," the outcome must store the **bare key string** (not token JSON) under the **target profile's credential slot** (= the profile id the broker reads).

`AuthOutcome` today (`daemon-host/src/auth.rs:72`): `{ credential_blob, credential_ref, account_label, transport_instance }`. `auth_complete` (`cred_auth.rs:59`) writes `store.set(credential_ref, credential_blob)` where `credential_ref = bind.credential_ref.unwrap_or(outcome.credential_ref)`, then (for a bind) attaches a `BoundAccount{transport_instance, credential_ref}` to the profile.

Two problems for the provider-key case: (a) the blob must be the bare key (descriptor handles this — `ProviderKey` sets `credential_blob = <key_field value>`); (b) the slot must be the **bound profile id**, and the node — not the client — must decide that.

**Design (node decides the slot):** add a non-wire field to `AuthOutcome`:
```rust
pub enum CredentialSlotKind {
    Derived,               // today: store under credential_ref (transport-account style); attach BoundAccount
    ProviderKeyForProfile, // model provider key: store under the BOUND PROFILE ID; NO BoundAccount
}
// AuthOutcome { .., pub slot: CredentialSlotKind }
```
`auth_complete` (`cred_auth.rs`) resolution rule:
- `Derived` → unchanged (bind.credential_ref override → BoundAccount attach). Generic `oauth2` + Matrix keep today's behavior byte-for-byte.
- `ProviderKeyForProfile` → require a `bind.profile`; `credential_ref = bind.profile.as_str()` (the model-key slot the broker reads); `store.set(profile_id, bare_key)`; **skip** the `BoundAccount` attach (a provider key is not a transport account). The `AuthCompleteResponse.credential_ref` reports the profile slot. This is exactly the pasted-key slot, so the minted key flows through `PooledStoreCredentialSource` → `Request.auth` unchanged.

`transport_instance`/`account_label` for provider-key outcomes: label `"openrouter"`; `transport_instance = provider/openrouter/<profile>` (informational; not used for provider-key slotting). See OQ-3 for the no-bind case (proposal: reject — provider-key minting is only meaningful bound to a target profile).
