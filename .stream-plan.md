New enum next to `ConnectionState` (~after line 2735):
```rust
/// Why a transport instance disconnected — a closed, Matrix-scoped set the node maps every
/// adapter-specific failure onto (the GError-shaped analogue). The node decides `fatal`; a thin
/// client keys re-auth affordances off `fatal`/`reason`, never off adapter strings.
#[cfg_attr(feature = "arbitrary", derive(arbitrary::Arbitrary))]
#[derive(Clone, Copy, Debug, PartialEq, Eq, Serialize, Deserialize)]
pub enum DisconnectReason {
    /// The operator/user asked to disconnect (`TransportDisconnect`, sign-out).
    UserRequested,
    /// A network/transport error (timeout, connection reset, homeserver unreachable).
    NetworkError,
    /// The stored credential was rejected (login/refresh failed).
    AuthenticationFailed,
    /// The server replaced this session with another client (soft-logout / device conflict).
    ReplacedByOtherClient,
    /// The instance config is invalid (bad homeserver URL, missing required field).
    InvalidSettings,
    /// A TLS/certificate validation failure.
    CertificateError,
    /// Any reason not captured above (the adapter's detail rides `message`).
    Other,
}
```

**Routing reconciliation design (node-owned, `is_self` removals):**
On a `MembershipChanged{ change ∈ {Left,Kicked,Banned}, is_self: true }` the node reconciles its
own routing registry BEFORE emitting the event (the event is only an invalidation pointer). The
routing pins are `ChatRoute`s keyed by `Origin` in the `chat_routes` table
(`daemon-store/src/sqlite.rs:66`, `routing_list`/`routing_get`/`routing_remove` at
`sqlite.rs:1497–1556`). The now-dangling origin is
`Origin{ transport, scope: OriginScope::Group{ chat: conv, thread: None } }`
(`daemon-protocol/src/lib.rs:946,975`). Reconciliation = look up the pin for that origin and
**drop it** (`routing_remove`) — the default policy — OR flag it (a soft "dangling" marker) if
the coordinator prefers reversibility. I recommend **drop** for v30 (mirrors `TransportRemove`
teardown; a re-join re-creates the pin on the next inbound). Mass-resync stays the existing
`ResyncNeeded` fallback (`lib.rs:3559`); the node emits `ResyncNeeded{scope:"roster"}` when a
whole-room resync is cheaper than per-member deltas. **Open question C** (drop vs. flag).

The reconciliation + the two emit seams live in the new
`crates/substrate/daemon-host/src/node_api/membership.rs`:
`emit_conversations_changed(transport, conv, change)`,
`emit_membership_changed(...)` (which calls `routing_remove` first on self-removal), both
routing through `node_feed()` (`internals.rs:504`, `NodeEventFeed::emit` at `internals.rs:282`).

### Item 9 — Structured foreign failure + mid-turn death (C6)

Exit-watch the foreign child: today the reader loop EOF at `foreign.rs:76` (`while let
Some(bytes) = reader.recv().await`) ends silently, so a client `poll`/`subscribe` hangs.
Synthesize `TurnFinished{Failed}` carrying `failure: Option<ForeignFailure{stage,agent}>` on

**Rust — `crates/contracts/daemon-protocol/src/lib.rs`:**
New types near `TurnSummary` (~before line 396):
```rust
/// Where a foreign agent failed (wire v30, C6) — the coarse stage the node attributes a foreign
/// child's death to, so a client can render "the agent crashed mid-turn" instead of hanging.
#[cfg_attr(feature = "arbitrary", derive(arbitrary::Arbitrary))]
#[derive(Clone, Copy, Debug, PartialEq, Eq, Serialize, Deserialize)]
pub enum ForeignStage {
    /// Failed launching / connecting to the foreign agent process.
    Spawn,
    /// Failed during the initialize/handshake.
    Handshake,
    /// Died mid-turn (stdout EOF while a turn was in flight).
    Turn,
    /// Stage not determinable.
    Unknown,
}

/// A structured foreign-agent failure carried on a terminal [`TurnSummary`] (wire v30, C6).
#[cfg_attr(feature = "arbitrary", derive(arbitrary::Arbitrary))]
#[derive(Clone, Debug, PartialEq, Eq, Serialize, Deserialize)]
pub struct ForeignFailure {
    /// The coarse failure stage.
    pub stage: ForeignStage,
    /// The foreign agent name (catalog id), when known.
    #[serde(default)]
    pub agent: Option<String>,
}
```
Extend `TurnSummary` (line 396) — serde-default so every existing encoding decodes:
```rust
    /// A structured foreign-agent failure (wire v30, C6). `Some` only on a foreign child's
    /// synthesized `TurnFinished{Failed}`; `None` for native turns and pre-v30 encodings.
    #[serde(default)]
    pub failure: Option<ForeignFailure>,
```
Update `TurnSummary::ended` (line 407) to set `failure: None`; grep every `TurnSummary { ... }`
literal (daemon-core, daemon-host, tests) — they must add `failure: None` (or adopt
`..TurnSummary::ended(..)`/a new `..Default` — `TurnSummary` does not derive `Default`, so I
recommend adding `failure: None` to each literal or a `TurnSummary::failed(ForeignFailure)`

```rust

        // synthesize a terminal TurnFinished{Failed} so a poll/subscribe consumer unblocks.
```

- **A (item 1):** `spawn_adapters` returns an opaque `Vec<JoinHandle>`. `transport_disconnect`

  `TransportId → (JoinHandle, adapter)` map so a single instance can be stopped/restarted?
